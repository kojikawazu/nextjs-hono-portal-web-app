# ドキュメント構成

`docs/` 配下のドキュメントが肥大化して読みにくくならないよう、ファイルサイズと分割方法を規定する。更新内容のルールは [documentation.md](./documentation.md)（影響マップ）を参照。

## ファイルサイズ上限

- `docs/` 配下の 1 ファイルは **150 行以内** とする。
- 追記により 150 行を超える場合は、内容を削るのではなく「分割」で対応する。

### 対象外

- **追記型の時系列ログは対象外**とする。具体的には監査レポート（`docs/security-audit-report*.md`）など、実施日ごとに末尾へ追記していく記録ドキュメント。これらは履歴の連続性を優先し、行数で分割しない。

## 150 行を超える場合の分割

1 ファイルで 150 行に収まらない場合は、**元ファイルと同名のフォルダを作成し、その中でファイルをトピック単位に分割する**。

- 例: `docs/07-api-specification.md`（150 行超） → `docs/07-api-specification/` フォルダを作成し、`gcs.md` / `mail.md` などに分割する。
- 分割後の各ファイルも **150 行以内** を守る。
- **元のファイル名はフォルダ名として維持する**（影響マップの逆引き対応を壊さないため）。

## index（readme.md）の付与

- 分割フォルダの直下には **`readme.md` を必ず追加** し、フォルダ内ファイルへの index（目次）とする。
- index には各ファイルへのリンクと、そのファイルが扱う内容の 1 行要約を記載する。

例（`docs/07-api-specification/readme.md`）:

```markdown
# API仕様書

| ファイル | 内容 |
|---|---|
| [gcs.md](./gcs.md) | GCS データ取得 API |
| [mail.md](./mail.md) | メール送信・CSRF API |
```
