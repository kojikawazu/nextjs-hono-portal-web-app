# 要求仕様書（Business Requirements）

## 1. プロジェクト背景

個人のポータルWebサイトをリニューアルするプロジェクト。旧サイトからの移行に伴い、以下の課題を解消することを目的とする。

- デザインの刷新（モダンなダークテーマUI）
- インフラの統一化（Vercel → Google Cloud Run）
- セキュリティの強化（Cloudflare導入）
- リソースの分離（Google Cloud Storage活用）

## 2. プロジェクトの目的・ゴール

- 個人の技術的な活動（ポートフォリオ、個人開発、サンプル開発）を一元的に公開するハブサイトを構築する
- 訪問者が開発者に直接問い合わせできるコンタクトフォームを提供する
- 外部リンク（GitHub、X、LinkedIn、ブログ）への導線を整備する

## 3. ステークホルダー

| 役割 | 説明 |
|------|------|
| 開発者（本人） | サイトオーナー兼開発者。コンテンツの管理・更新を行う |
| 訪問者 | ポートフォリオや開発履歴を閲覧し、問い合わせを行うユーザー |

## 4. スコープ

### スコープ内

- ホームページ（ヒーローセクション + ナビゲーション）
- 個人開発履歴の一覧表示ページ
- サンプル開発履歴の一覧表示ページ（画像付き）
- お問い合わせフォーム（入力→確認→送信完了の3ステップ）
- メール送信機能（Resend経由）
- GCSからのデータ取得API
- 外部リンク（ポートフォリオ、ブログ、GitHub、X、LinkedIn）への導線
- レスポンシブデザイン（モバイル対応）
- Cloud Runへのデプロイ（Docker + GitHub Actions）

### スコープ外

- ユーザー認証・ログイン機能
- コンテンツ管理システム（CMS）
- データベース
- 多言語対応
- コメント・フィードバック機能

## 5. 制約

- 個人プロジェクトのため、コスト最小化を優先
- コンテンツデータはGCS上のJSONファイルで管理（DB不使用）
- メール送信にはResendの無料枠を使用
- ドメイン: smartportalcom.com
